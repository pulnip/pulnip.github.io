## 레이어 구조

```text
┌─────────────────Layer 3: Application──────────────────┐
│                          App                          │
└───────────────────────────────────────────────────────┘
                            ↓
┌──────────────Layer 2: High-Level Systems──────────────┐
│       Render   Input   Physics   Script   Audio       │
│        [✓]      [ ]      [ ]      [ ]      [ ]        │
│  [✓] Resource (→ assimp, toml++, stb)                 │
│  [✓] Scene - Specification Assembler                  │
└───────────────────────────────────────────────────────┘
                            ↓
┌───────────────────Layer 1: Platform───────────────────┐
│  [ ] RHI ──┬── [✓] MetalRHI (Apple)                   │
│            ├── [ ] D3D12RHI (Windows)                 │
│            └── [✓] NullRHI  (Test)                    │
│  [✓] Platform (→ SDL3)                                │
└───────────────────────────────────────────────────────┘
                            ↓
┌────────────Layer 0: Core (Pure Libraries)─────────────┐
│            EngineCore (→ Core)   ECS  Log             │
└───────────────────────────────────────────────────────┘
```

내가 생각하는 기본적인 의존성 방향은 위와 같다. 레이어 구조는 그걸 사용해도 되는지를 알려주는 역할이다. 그리고 한 레이어 내에서는 모듈로, 모듈은 그 부품이 어떤 역할을 담당하는지 보여준다.

## 오늘의 생각
