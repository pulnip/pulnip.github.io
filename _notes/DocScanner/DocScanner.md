## 기본-개념들

(A) 한 프레임 내의 모든 픽셀은 <ins>페이지 픽셀</ins> $p_i$ 이거나, <ins>배경 픽셀</ins> $b_i$ 이다. (exclusive)
페이지 픽셀이면서 배경 픽셀인 픽셀은 존재하지 않는다. $\nexists i,p_i=b_i$
페이지 픽셀이 만드는 영역 $p=\bigcup{p_i}$ 을 <ins>페이지 영역</ins>이라고 한다.
배경 픽셀이 만드는 영역 $b=\bigcup{b_i}$ 를 <ins>배경 영역</ins>이라고 한다.

(B) 각 프레임은 <ins>펼친 상태</ins> $s_i^{(o)}$이거나, <ins>넘기는 상태</ins> $s_i^{(f)}$이다.
펼친 것도 넘기는 것도 아닌 상태는 존재하지 않는다. $S = \{s^{(o)}, s^{(f)}\}$
펼친 상태와 넘기는 상태는 해당 페이지를 넘긴 정도에 따라 결정된다. $P(s_i^{(o)})+P(s_i^{(f)})=1$
또한, 펼친 상태에서 넘기는 상태로, 넘기는 상태에서 펼친 상태로의 전이는 **부드럽게** 일어난다.

![](assets/img/graph.jpg)

(C) 펼친 상태에 있는 프레임과 그 인근 프레임을 <ins>페이지 블럭</ins>이라고 한다.
모든 프레임은 페이지 블럭들 중 하나에 그리고 하나에만 속한다.

## 기본 가정들

(가) 동영상의 각 프레임마다 스캔 대상이 되는 책이 존재하며, 단 하나이다.
(나). 동영상 내에서 책을 넘기는 모션만 있다.
	책 자체를 이동시키는 모션 등은 없다.
(다). 동영상은 바닥에 수직하게 책을 향하여 촬영된다.
	즉, 어떤 프레임이 *펼친 상태*에 있으면, 해당 프레임 내 *페이지 영역의 모양*은 <ins>직사각형</ins>이다.
(라). 페이지를 넘기는 속도는 일정하며, fps이하이다.
	즉. 페이지를 넘기는 속도는 정규분포를 따른다.

(라). 한 프레임 내의 정보(에지, 색공간)만으로 페이지 픽셀과 배경 픽셀을 구분할 수 있다.
	한 프레임 내의 정보만으로 페이지 영역의 모양을 알 수 있다.
(마). (라)가 성립하지 않을 시, 모션이 발생하지 않는 영역이 배경 영역이다.

### 1-(나)의 주요 문제점

<ins>1-(나)-P1. 한 프레임만으로는 페이지 영역-배경 영역 분리 불가능하다.</ins>

예시 상황들
- **정반사**에 의해, 배경 영역 일부의 밝기가 더 클 수 있다.
- **그림자**에 의해, 페이지 영역 일부의 밝기가 더 작을 수 있다.
- **정반사**+**그림자** 두 상황이 같이 나타날 수 있다.

가능한 해결 방법
- 프레임 차분(frame difference)을 통해 배경 영역을 추출한다.

<ins>1-(나)-P2. 실제 페이지 영역의 채도가 배경에 비해 높다.</ins>

예시 상황들
- 페이지 안에 그림이 있다.
- 페이지나 책의 색상이 빨간색(채도가 높은 경우)이나 검은색(밝기가 낮은 경우)등, 흰색이 아니다.

<ins>1-(나)-P3. 실제 페이지 영역의 크기가 영상 내에서 그다지 크지 않다.</ins>

예시 상황들
- 책 옆에 직사각형 모양의 큰 종이가 있다.

### 1-(나)의 실제 프로세스

<ins>페이지 영역 필터링 과정</ins>

전역 처리
1. global thresholding: "영상 평균 밝기"의 "50%" 이하
	평균 밝기에도 한참 못 미치는 픽셀 필터링
2. (밝기 기준) 픽셀 clustering: k-means clustering
	k = 밝기 히스토그램 상 "peak" 수 (단, "4"이하)
3. 평균 밝기가 큰 클러스터를 페이지 영역 후보로 간주한다.
	(정반사에 의한 밝기 고려, 실제로는 평균 밝기 상위 1, 2를 사용.)
4. 페이지 영역=1, 배경 영역=0인 마스크 획득.

지역 처리
1. 형태학적 연산 수행.
	- 너무 작은 페이지 영역 후보 픽셀 삭제 (∵ 페이지 영역이 아님)
	- 페이지 영역의 경계 픽셀 스무딩.

## 2. 직사각형도 추출

추가 가정2: 추출된 페이지 영역 후보 중, 가장 큰 영역이 실제 페이지 영역일 것이다.

페이지 영역으로부터, 아래의 직사각형도(Rectangularity) 정의에 따라 계산한다.
$$\text{직사각형도}=\frac{\text{페이지 영역의 넓이}}{\text{영역을 감싸는 가장 작은 직사각형의 넓이}}\space(\le1.0)$$

### 직사각형도 특징의 주요 문제점들

<ins>2-P1. (펼친 상태에서) 직사각형 영역 중 페이지 영역이 가장 크진 않다.</ins>
<ins>2-P2. (펼친 상태에서) 페이지 영역의 모양이 사실은 직사각형이 아니다.</ins>

예시 상황들
- 배경에 직사각형의 물체가 있으며, 페이지 영역보다도 크다.
- 손이나 옷, 혹은 다른 물체에 의해 페이지 영역이 가려졌다.
- (카메라가 수직이 아니어서) 원근에 의해 사다리꼴 등으로 모양이 왜곡된다.

## 3. 페이지 블럭 별 프레임 구분

(다)에 의해, 페이지를 넘기는 것은 곧 펼친 상태의 (직사각형) 모양에 변형을 가하는 것이다.
따라서, 페이지 블럭 간 경계에서 직사각형도는 골짜기 모양을 이루면서 변화해야한다.

(라)에 의해, (골짜기) 특징 간 시간적 거리가 너무 가까울 경우, 잘못 추출된 것이다.

{% mermaid %}
graph TD
	RAW(전체 프레임)
	IN(직사각형도 시계열)
	subgraph 프레임 구분
		A[스무딩]
		B[골짜기 특징 추출]
		C[페이지 넘김 시각 추정]
		D[페이지별 프레임 구분]
	end
	OUT(구분된 프레임들)

	IN --> A --> B --> C --> D --> OUT
	RAW --> D
{% endmermaid %}

### 골짜기 특징의 주요 문제점들

<ins>3-P1. 시간 축에 대한 특징의 크기가 너무 크고 작거나,
심지어는 (실제로는 다른 페이지로 구분이 됨에도) 특징이 발견되지 않을 수 있음.</ins>

(추가 설명) 골짜기 모양에 대한 시계열 데이터를 봤을 때
골짜기의 대략적인 크기를 알 수 있다.
(이상적 상황에서) 3s~5s에 골짜기 모양이 나타났다면,
골짜기의 크기는 5s-3s=2s라고 할 수 있다.

실제 예시
- 사용자가 책을 너무 빠르게 넘김. (특징의 크기 작아지거나 소멸)
- 사용자가 책을 너무 느리게 넘김. (특징의 크기가 커짐)

가능한 해결 방법
- (실행 중, 페이지 번호 부여 전) 앱으로 사용자에게 넘기기 적절한 타이밍을 알려준다.
- (실행 후, 페이지 번호 부여 후) 골짜기 모양이 약할 때의 구분은 사용자에게 맡긴다.
- (실행 후) 각 페이지 블럭 별 대표 프레임의 유사성을 비교한다.
	3번 블럭(30~38번 프레임)의 대표 프레임인 34번 프레임은
	2, 4번 블럭의 대표 프레임과 유사할 가능성이 있다.
	(2번과 3번 블럭은 사실, 같은 2번 페이지일 수 있으므로)

### 3의 실제 프로세스

입력: 직사각형도 시계열 데이터

<ins>스무딩</ins>

- (시간 축에 대해) 가우시안 커널로 1D 컨볼루션.
	kernel_size=7, duplicate padding

<ins>골짜기 특징 추출</ins>

- 골짜기 커널로 1D 컨볼루션
	\[1, 2, 3, -12, 3, 2, 1\], duplicate padding
- 특징 필터링
	$ReLU(x-0.2)>0$ (0.2는 실험적)

<ins>페이지 넘김 시각 추정</ins>

- 일정한 chunk를 이루는 특징에 대해, chunk 중간을 페이지 구분 후보로 간주.
- ((c)-1에 의해) 두 후보 간 거리가, 구분 후보 간 평균 거리의 절반에도 못 미칠 경우 두 후보 병합.

> 참고: 데이터가 *모두 수집된 후*가 아닌,
> *새로운 프레임이 들어올 때*마다 컨볼루션 수행하도록 변경 가능.

## 4. 페이지 블럭 별 대표 프레임 선택

페이지 블럭 내 가운데 프레임을 대표 프레임으로 선택한다.

### 4의 주요 문제점

<ins>4-P1. 너무 단순한데, 이것만으로 충분한가?</ins>
